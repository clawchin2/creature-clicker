# Memory Log - 2026-03-02

## Key Learnings & Decisions

### CORE FIRST, POLISH LATER (Established 2026-03-02)
**Decision:** Build minimum viable features first, add polish only after core works.

**Implementation Order:**
1. Core functionality (working button, basic click)
2. Verify core works
3. Add polish (animations, sounds, effects)

**Why:** Prevents agent timeouts, faster iterations, working game at each step.

---

### Protocol Violations (2026-03-02)
**Issue:** Multiple violations where I fixed code directly instead of using agents.

**Root Cause:** Pressure to deliver, old habits, underestimating complexity.

**Impact:** Wasted API credits, broken builds, user frustration.

**Solution:** Hard rule - ALL code changes go through agents, no exceptions.

---

### Roblox UI Development Pattern

**Working Structure:**
```
StarterPlayer/StarterPlayerScripts/
├── ClickerUI.client.lua (LocalScript)
└── HatchShopUI.client.lua (LocalScript)
```

**Critical Requirements:**
- Must use `.client.lua` extension (becomes LocalScript)
- Remote folder must be named exactly: "CreatureClickerRemotes"
- PlayerGui access must have timeout: WaitForChild("PlayerGui", 30)
- No complex StarterGui JSON in project.json

---

### Common Agent Failures

**What Fails:**
- Complex UI with animations in one task
- Multiple features in one agent
- Timeout set too low (5m too short for complex UI)

**What Works:**
- Single feature per agent
- Core functionality first (no polish)
- Simple, direct implementations
- Validation before next step
